# 完成应用配置提取字段管理功能

## 现状分析
1. 后端 `AppConfig` 模型已包含 `extraction_fields` 字段，支持动态提取配置
2. 记忆抽取逻辑 `extract_elements` 已实现基于 `extraction_fields` 的动态 prompt 构建
3. 前端 HTML 已添加提取字段配置 UI，但 JavaScript 逻辑未完成
4. 缺少动态字段管理、加载和保存逻辑

## 实施计划
1. **更新 `app/static/js/config.js`**：
   - 添加 `addExtractionField` 事件监听，实现动态添加提取字段
   - 添加 `remove-field` 事件监听，实现动态删除提取字段
   - 更新 `loadAppConfig` 函数，添加提取字段列表填充逻辑
   - 更新 `saveAppConfig` 函数，添加提取字段数据收集逻辑
   - 初始化提取字段管理功能

2. **测试流程**：
   - 启动应用服务
   - 访问配置编辑页面
   - 动态添加/删除提取字段
   - 保存配置并验证数据库更新
   - 提交聊天历史，验证记忆抽取是否使用新配置
   - 查询记忆，验证提取要素是否正确

## 预期效果
1. 前端可以可视化配置不同应用的提取字段
2. 提取字段配置会影响记忆抽取逻辑
3. 记忆抽取使用基于配置的动态 prompt
4. 整个系统支持 app 级别的个性化记忆管理

## 技术要点
- 动态 DOM 操作，实现提取字段的增删改查
- JSON 数据处理，将表单数据转换为 API 可接受格式
- 事件驱动编程，处理用户交互
- 前后端数据一致性验证
