# 功能整合与路由优化计划

## 1. 问题分析
当前系统存在功能重复问题：
- `AppConfig`和`BusinessTemplate`功能重叠，都包含提取模板、提取字段等配置
- 存在两套相似的CRUD API，增加了系统复杂度
- 违反了"少即是多"的设计原则

## 2. 整合方案
### 2.1 核心整合思路
- **保留`AppConfig`**：作为主要的应用配置模型
- **合并`BusinessTemplate`功能**：将模板的核心功能整合到`AppConfig`中
- **删除冗余API**：移除`BusinessTemplate`相关的API路由
- **简化前端调用**：统一使用应用配置API

### 2.2 具体整合步骤
1. **删除模板相关API路由**：
   - 删除`app/api/memory/templates.py`文件
   - 从`app/api/memory/__init__.py`中移除模板路由的引用

2. **调整内存优先级API**：
   - 评估内存优先级功能的必要性
   - 考虑将其功能合并到应用配置中
   - 或保留但简化API设计

3. **更新前端代码**：
   - 移除模板管理相关的前端代码
   - 简化配置管理界面
   - 确保所有功能使用统一的API

4. **测试功能完整性**：
   - 测试核心API路由
   - 确保整合后功能正常
   - 验证前端适配情况

5. **提交代码到git**：
   - 提交整合后的代码
   - 使用清晰的提交信息
   - 记录整合的主要变更

## 3. 预期效果
- **减少API端点数量**：删除重复的模板API
- **简化系统设计**：遵循"少即是多"原则
- **统一配置管理**：所有配置通过应用配置API管理
- **降低维护成本**：减少了需要维护的代码和API
- **更好的用户体验**：用户只需要使用一套API

## 4. 实施顺序
1. 删除模板相关API路由
2. 优化内存优先级API（可选）
3. 更新前端代码
4. 测试所有功能
5. 提交代码到git

## 5. 注意事项
- 确保整合后所有功能正常工作
- 保持向后兼容性（如果有现有客户端）
- 清晰记录整合的变更
- 遵循代码质量和架构原则
